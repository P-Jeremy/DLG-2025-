# ADR 0004 - Utilisation des assistants IA dans le développement

## Statut

Accepté

## Date

2026-03-30

---

## Contexte

Le projet est développé avec l’aide d’assistants IA modernes, notamment :

* Claude Code
* GitHub Copilot

Ces outils font partie intégrante de mon environnement de développement et n'influencent pas la manière dont je conçois et produis le code.

---

## Décision

J’intègre les assistants IA comme des outils de support au développement.

Ils sont utilisés pour :

* accélérer l’écriture de code
* proposer des solutions techniques
* générer des tests
* améliorer la qualité globale

---

## Règles d’utilisation

* Je reste responsable du code produit
* Le code généré est systématiquement relu et validé
* Les décisions d’architecture ne sont pas déléguées à l’IA
* L’IA est utilisée comme un support, pas comme une source de vérité

---

## Raisons

* Gain de productivité
* Accélération de l’apprentissage
* Aide sur les tâches répétitives
* Exploration rapide de solutions

---

## Conséquences

### Avantages

* Développement plus rapide
* Réduction des erreurs simples
* Amélioration continue du code

---

### Inconvénients

* Risque de dépendance
* Nécessité de validation humaine systématique
* Possibilité de propositions inadaptées

---

## Décision finale

Les assistants IA sont utilisés comme un levier d’accélération du développement, tout en conservant un contrôle humain sur la qualité et les décisions techniques.
